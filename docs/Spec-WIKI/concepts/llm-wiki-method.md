# LLM Wiki method (Karpathy)

**Тип:** concept
**Статус:** stable
**Источники:** [overview §4](../raw/20260507-ai-steward-wiki-only-overview.md), [Karpathy gist](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f)

## Трёхслойная архитектура

1. **Raw Sources** (`raw/`) — неизменяемые исходники: PDF, статьи, картинки, выписки. LLM только читает.
2. **Wiki** — Markdown-страницы, генерирует и поддерживает только LLM: саммари, entity/concept-страницы, перекрёстные ссылки.
3. **Schema** — `CLAUDE.md` в корне папки. Конституция вики.

## Обязательные файлы

1. `CLAUDE.md` — schema/конституция.
2. `index.md` — каталог страниц с однострочниками.
3. `log.md` — append-only хронология (`## [YYYY-MM-DD] op | Title`).

## Три операции

1. **Ingest** — обработка `raw/` по одному файлу: саммари → обновить entity/concept → запись в `log.md` + `index.md`.
2. **Query** — поиск по существующим страницам, синтез ответа с цитатами, **сохранение** ответа обратно в вики.
3. **Lint** — периодический аудит: противоречия, осиротевшие страницы, устаревшие утверждения, пробелы.

## Ключевая идея

Вики — **persistent, compounding artifact**. Каждый запрос конденсируется обратно в файлы. База растёт и становится умнее. Не нужен RAG / векторная БД — достаточно файловой системы и дисциплинированного агента.
