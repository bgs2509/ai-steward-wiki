# Schedule Profiles

**Тип:** concept
**Статус:** draft
**Источники:** диалог 2026-05-08, [time-tracker](../entities/time-tracker.md)

## Суть

Расписание периодических опросов трекера зависит от **типа дня**. Будни и выходные — разный шаг. Окно дня жёстко фиксировано: 06:00 (подъём) → 23:00 (отбой).

## Профили

1. **Weekday (будни, пн–пт по умолчанию)**
   1. Шаг опроса: **120 минут**
   2. Слоты: 06:00 (подъём), 08:00, 10:00, 12:00, 14:00, 16:00, 18:00, 20:00, 22:00, 23:00 (отбой)
2. **Weekend (выходные, сб–вс по умолчанию)**
   1. Шаг опроса: **300 минут**
   2. Слоты: 06:00 (подъём), 11:00, 16:00, 21:00, 23:00 (отбой)
3. **Holiday (праздники)**
   1. По умолчанию = weekend.
   2. Источник списка праздников — открытый вопрос.

## Границы окна

1. **06:00** — фиксированное сообщение «подъём». Не опрос, не требует ответа.
2. **23:00** — фиксированное сообщение «пора спать». Аналогично.
3. **Вне окна (23:00–06:00)** — никаких сообщений.

## Day-override

Если на слот стоит planner-семантичный item в `jobs.db` — он **бьёт** профильный слот:
1. Юзер указал «22:00 пораньше спать» → стандартное «пора спать в 23:00» не отправляется.
2. Юзер указал «09:00 работа» → опрос «что делал в 08:00» отправлен как обычно, опрос «10:00» подавляется (`jobs.db` item в окне ±30 мин — anti-spam, см. [mandatory-checkins](mandatory-checkins.md)).

## Конфигурация (юзер-уровень)

В `{User}/CLAUDE.md`:
1. `tracker.window`: `"06:00-23:00"`
2. `tracker.weekday_step_min`: `120`
3. `tracker.weekend_step_min`: `300`
4. `tracker.weekend_days`: `["sat", "sun"]`
