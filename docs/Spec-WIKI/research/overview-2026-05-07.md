# Summary: AI Steward Wiki-Only Overview (2026-05-07)

**Тип:** research
**Статус:** stable
**Источник:** [raw/20260507-ai-steward-wiki-only-overview.md](../raw/20260507-ai-steward-wiki-only-overview.md)

## Суть

Изолированный сервис на отдельной VPS, превращающий Claude Code CLI в персонального WIKI-ассистента. Multi-tenant Telegram-бот: юзер кидает любой контент (текст/файл/фото/голос), бот идентифицирует по `telegram_id`, классифицирует и направляет в персональную WIKI-папку. Karpathy LLM Wiki как единственный режим работы.

## Ключевые архитектурные решения (из overview)

1. **Двухступенчатый запуск Claude** (§2.1): первый вызов — классификатор (выбор WIKI + сессии), второй — исполнитель в выбранной WIKI. См. [two-stage-launch](../concepts/two-stage-launch.md).
2. **Sibling-only домены** (§3a): `USERS/<NAME>/<Domain>-WIKI/` параллельно, без вложенности. См. [sibling-only-domains](../concepts/sibling-only-domains.md).
3. **Anti-nesting** (§5): WIKI внутри WIKI запрещены. См. [anti-nesting](../concepts/anti-nesting.md).
4. **Smart Inbox + Router** (§8.3.1): `Inbox-WIKI/` как единая точка входа, Router-Claude классифицирует и предлагает действия. См. [smart-inbox-routing](../concepts/smart-inbox-routing.md), [inbox-wiki](../entities/inbox-wiki.md).
5. **Унифицированный Job-объект** (§8.3.2): `reminder_job` / `wiki_job` / `digest_job` в одной таблице с дискриминатором `kind`. См. [job-model](../entities/job-model.md).
6. **Двухуровневый intent-detection** (§8.3.3): Haiku fast-path для тривиальных intent → reminder_job напрямую; CLI только для wiki-операций.

## Стек

Python 3.11 + uv, aiogram 3.x, Claude CLI (asyncio.subprocess), APScheduler/crontab, pydantic-settings, SQLAlchemy 2.x async + SQLite, systemd.
