# Completion report — Inbox-WIKI Phase-D.b.1: `digest_job` vertical slice (recurrence parsing → CronTrigger, direct-fire under run_wiki_session's lock, pipeline confirm flow)

## Goal

Let a user create a recurring digest by natural language («присылай сводку каждый день в 9», «сводка по будням в 19:00») via an inline-button confirm; on schedule, run Claude with `--add-dir` into the owner's `<Name>-WIKI`s and deliver the summary to Telegram. Split out of `aisw-19o` (too large for one context window); the D-024 presentation polish (cards, TL;DR-as-section, 4096 section-split, `/expand`, `/digest_now`) and a real `jobs.db` planner-context query are `aisw-w3k` (Phase-D.b.2); the asyncio.PriorityQueue worker-loop consumer the original `aisw-19o` description bundled in is de-scoped out of `aisw-19o` entirely (YAGNI — no consumer today).

## Design decisions (auto-approved gates; see ADR-007)

Direct-fire under `run_wiki_session`'s own LockAcquirer, **not** the PriorityQueue producer/consumer (YAGNI — no consumer today; PriorityQueue worker-loop de-scoped out of `aisw-19o`) · `Recurrence` as a reusable frozen Pydantic value in a new `classifier/recurrence.py`, **not** an extension of `time_parse.py` · rule-based ru recurrence parser, Haiku fallback stubbed · `DigestPayload` freely replaced (no producer existed); no schema migration; recurrence trigger owned by `SQLAlchemyJobStore`, `jobs.Job` is the domain SSoT · separate `set_digest_context` registry carrying the `scheduler` handle (more deps than the reminder one) · WIKI-set = `'all'` sentinel (glob `<owner>/*-WIKI` minus `Inbox-WIKI`); lock the primary only · plain `sender.send_message` delivery this phase (D-024/D-025 presentation + `deliver_output` audit row → `aisw-w3k`) · 3-strike auto-disable + DLQ (D-019; 600s timeout = strike); no retry-with-backoff.

## Deviations from the plan

- **Delivery** — the plan/design recommended `tg/output.deliver_output` (audit `run_outputs` row + `ChainSplitter`/`send_document`); implemented as a plain `sender.send_message` with a ~4000-char truncation to keep this phase's `set_digest_context` surface small. Switching to `deliver_output` is a small follow-up tracked under `aisw-w3k` (which already owns the D-024 presentation work). (Auto-approved deviation.)
- **Planner-semantics context** — the design called for a `jobs.db` query of the owner's upcoming rows in the window; implemented as a one-line `f"Окно сводки: ближайшие {window_hours} ч."` note for the MVP. The real query → `aisw-w3k`. (Auto-approved deviation.)
- **`set_digest_context` shape** — dropped the `Semaphore`/`WikiLockManager` from the registry (per the plan's flagged executor decision) — `run_wiki_session`'s acquirer already does semaphore→memlock→flock, so `fire_digest_job` would have double-locked. The registry carries `scheduler` instead (for `remove_job` on auto-disable).
- **`prompts/recurrence.md`** — shipped but not wired (the rule-based parser covers the MVP phrasings; a Haiku fallback is a later wiring).
