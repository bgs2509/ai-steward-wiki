### Critical

1. **[Spec↔code] AM/PM 12-hour corruption in NL recurrence — `src/ai_steward_wiki/classifier/recurrence.py:186`**
   - Evidence: `if part in {"вечера", "ночи", "дня"} and h < 12: h += 12`
   - Why: `"в 2 ночи"` → `14:00`, `"в 3 ночи"` → `15:00` (off by 12h); `"12 ночи"` (midnight) stays `12:00` (noon); `"10 дня"` → `22:00`. Any recurring daily/weekly/monthly cron built from these phrasings fires **12 hours off, every cycle** — silent, persistent wrong-time delivery.
   - Fix: replace the blanket `+12` with a per-part AM/PM map. Add boundary unit tests.

2. **[Spec↔code/concurrency] Reminder done/skip does not suppress the scheduled fire → double delivery — `src/ai_steward_wiki/scheduler/firing.py:235`**
   - Evidence: guard is `if job is None or job.status != "pending"`; done/skip handler `src/ai_steward_wiki/tg/callbacks.py:165-187` only calls `_cas_set_state` on `user_state`, never touches `status` nor cancels the APScheduler `reminder:<job_id>` trigger.
   - Why: a user tapping done/skip on a card for a **not-yet-fired** reminder leaves `status='pending'` → the reminder still fires after the user already resolved it (zombie/duplicate delivery).
   - Fix: on done/skip, also drive `Job.status` terminal and/or have `fire_job`'s guard also skip when `user_state ∈ {done, skipped}`.

3. **[Security] argv flag injection into the cron-user Claude CLI — `src/ai_steward_wiki/scheduler/consumer.py:301`**
   - Evidence: `argv = ["systemd-run", ..., "--", binary, *system_prompt_argv(self._prompt_path), msg.command]` — the lone `--` terminates `systemd-run`'s parsing, not `claude`'s; `msg.command` (raw user `/cron_add` text) is the trailing argv element with **no `--` before it**.
   - Why: a command starting with `-` is parsed by the Claude CLI as a **flag**, not a prompt. This path applies none of `runner.py`'s guards, so an injected `--dangerously-skip-permissions` / `--add-dir /` widens the CLI's tool/permission surface — defeating the per-job sandbox.
   - Fix: insert a literal `"--"` immediately before `msg.command`; bring this path to parity with `runner._build_argv`.
