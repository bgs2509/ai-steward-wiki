# Completion report — Inbox-WIKI Phase-D.a: reminder_job — cron bridge (Stage-0 fast-path → confirm → APScheduler DateTrigger → TG deliver)

## Goal

Let a user create a one-shot reminder by natural language («разбуди завтра в 6 позвонить врачу», «напомни через час…») via an explicit inline-button confirm, and deliver it at that time as a plain Telegram message (no Claude, no WIKI). Recurring digests («каждый день в 9 сводка») are out of scope → answered with a ru "not yet" line; the real digest support is `aisw-19o` (Phase-D.b).

## What changed

1. **New module `M-SCHEDULER-FIRING` (`src/ai_steward_wiki/scheduler/firing.py`)** — `set_firing_context(*, sender, jobs_session_maker)` installs a module-level `(TgSender, async_sessionmaker)` registry; `create_reminder_job(session, scheduler, *, owner_telegram_id, chat_id, when_utc, message, lead_time_min=0, correlation_id="") -> int` INSERTs+**commits** a `jobs.Job(kind='reminder_job', status='pending', priority=Lane.USER_WRITE, scheduled_at_utc=<naive UTC>, payload=ReminderPayload(...).model_dump(mode="json"))` then `scheduler.add_job(fire_job, trigger=DateTrigger(run_date=when_utc), args=[job_id], id=f"reminder:{job_id}", misfire_grace_time=None)` (commit-before-add_job ordering — a crash in the gap leaves at worst a pending row with no trigger; no MVP reconciliation), logs `scheduler.reminder.scheduled`; `fire_job(job_id: int)` (the picklable APScheduler callback) raises `FiringNotInitialisedError` if the registry is unset, else loads the `Job`, **guards `status == 'pending'`** (else `scheduler.reminder.skipped`), marks `in_progress`+`started_at_utc`, `await sender.send_message(chat_id, "🔔 Напоминание: <message>")`, marks `done`+`finished_at_utc` (or `failed`+`last_error` + `scheduler.reminder.deliver_failed` on a send error — no retry/DLQ for reminder_job), logs `scheduler.reminder.fired`/`.delivered`.

## Design decisions (auto-approved gates; see ADR-006, design frontmatter T-1..T-8)

Stage-0 fast-path detection, **not** the Stage-1a router (tech-spec §6; the bd title's "RouterDecision-carries-reminder" read as intent, not a literal field) · explicit inline-button confirm reusing the Phase-C machinery, **not** implicit-ack · `jobs.jobs` is the domain SSoT, APScheduler's `SQLAlchemyJobStore` holds the trigger, commit-before-add_job, **no** MVP reconciler · `fire_job` is a stable module-level callable taking only a picklable int, context from a module-level registry (differs from `maintenance.py`'s re-registered-each-boot non-picklable args) · `prefer_future=True` + user TZ from `UserRecord.tz` | `Settings.default_user_tz` · `reminder_text` read opportunistically, `prompts/classifier.md` **not** bumped · `misfire_grace_time=None` + `EVENT_JOB_MISSED` listener + `coalesce` (no DLQ for reminder_job) · recurring-digest phrasing stubbed to `aisw-19o`.
