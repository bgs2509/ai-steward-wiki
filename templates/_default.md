# Default WIKI

> Универсальная WIKI без специализации домена. Используется, когда контент не подходит
> ни под одну предметную область.

## Format

1. Короткие mobile-friendly ответы.
2. Списки вместо таблиц.
3. Длинные ответы (>3500 символов) — суммаризировать; полный текст приложением (D-025).

## Data layout

1. Структурированные данные (числа, измерения, суммы, даты) — в CSV соответствующего раздела (новая строка в существующий файл).
2. Нарративные заметки — в markdown-страницу по теме (одна тема — один файл).

## File resolution

1. Найди существующий подходящий файл и **допиши / отредактируй** его; новый создавай только если подходящего нет.
2. Не создавай произвольные `pages/` или иные структуры в обход разделов выше.

## Pre-flight

Для важных операций (создание / удаление / переименование / merge / split / edit rules) —
5 шагов: intent-grounding → blind-spot scan → clarification → confirm → execute+audit (D-041).

## Inbox hint

intents: append_data, query, page_create, page_edit
keywords: общее, заметка, разное, прочее
priority: 10
